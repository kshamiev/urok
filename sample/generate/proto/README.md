### generate

Инструмент по автоматизации сопоставления типов для GRPC

Генерация описаний прототипов, самих прототипов и методов конвертации типа в обе стороны.
- Инструмент можно встроить в уже существующий проект или репозиторий.
- Обрабатываются только публичные и помеченные тегом json поля.
- Описание ФС инструмента:


    generate/proto/main.go    программа генерации, (глобальные настройки)
    typs/adv.go         регистрация типов для обработки и кастомных функций обработки их свойств

Из коробки обрабатывает:
+ базовые типы: `bool, string, int..., uint..., float..., []byte, []string`
+ `typ.UUID` - реализация работы с полями UUID
+ `time.Time` - дата и время
+ `time.Duration` - время
+ `decimal.Decimal` - работа с дробными числами
+ ENUM в парадигме GRPC (`enum.*`)
+ ссылки на другие типы в этом же пакете (`typs`)
+ срезы ссылок на другие типы в этом же пакете (`typs`)
+ имеет спецификацию работы с типами используемыми в библиотеке boiler
(`null.JSON, null.Bytes, null.String, null.Time, types.StringArray`)

####  Использование
- Клонируем и создаем свою репу. Переопределяем пакеты при желании (в этом случае правим код). 
- В пакете `typs` создаются необходимые типы (golang).
(пакет `typs` можно переименовать, но придется немного поправить код генератора) 
Для обработки свойств пользовательских типов нужно реализовать их функции генерации (`typs/adv.go`)

Пример создаваемого типа:

```
type Areass struct {
	ID                 typ.UUID  `json:"id"`
	FederalDistrictID  typ.UUID  `json:"federal_district_id,omitempty"`
	Name               string    `json:"name"`
	UtcHours           int       `json:"utc_hours"`
	City               *City     `json:"city"`
	Users              []*User   `json:"users"`
	CreatedAt          time.Time `json:"created_at"`
	UpdatedAt          time.Time `json:"updated_at"`
	DeletedAt          null.Time `json:"deleted_at,omitempty"`
}

AreaSlice []*Area
 ```
- Далее заходим в `typs/adv.go` и добавляем в конфигурацию необходимые нам типы.
- После конфигурирования запускаем в консоли команду ```make pb```.
- Все наши типы обработаны и подготвелны к работе. 
- Сгенерированный протофайл с прототипами подключается в реализуемые далее сервисы.

Примечание:

- Под каждый указанный тип создастся описание прототипа, сам прототип и на него (тип) будут созданы методы конвертации (сопоставления).
- Если была допущена ошибка и файл с методами был сгенерирован битый.
После исправления ошибки перед запуском генератора удалите этот файл (`typs/advanced.go`).
